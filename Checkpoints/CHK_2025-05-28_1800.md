## 2. Snapshot Técnico (Para Continuidad de IA)

### 2.1. Descripción Funcional

Se ha implementado un patrón de edición consistente en toda la aplicación. Cada elemento en las listas de paneles principales (Clientes, Candidatos, Puestos, Procesos) ahora muestra un icono de edición. Al hacer clic, se reutiliza el modal de creación existente, pero se pre-cargan los datos del documento seleccionado desde Firestore. La lógica de guardado es ahora condicional: utiliza `update()` si se está editando un documento existente o `add()` si se está creando uno nuevo. Este enfoque sigue el principio DRY (Don't Repeat Yourself) y mantiene el código limpio.
